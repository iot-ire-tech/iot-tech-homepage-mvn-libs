decode

  • decode(source: Blob | Buffer | ArrayBuffer | Uint8Array | Uint8ClampedArray | HTMLImageElement | HTMLCanvasElement | HTMLVideoElement | string): Promise<TextResult[]>
    • Defined in dbr-wasm.d.ts:96

    The main decoding method can accept a variety of data types, including binary data, images, base64, urls, etc. In the nodejs environment:

      The method can accept Buffer, Uint8Array,

      base64 with mimetype, disk relative paths, disk pair paths, and absolute URLs.

      All data should be encoded as either jpg, png, bmp or gif.

    In the browser environment:

      The method can accept Blob, ArrayBuffer,

      Uint8Array, Uint8ClampedArray, HTMLImageElement, HTMLCanvasElement, HTMLVideoElement,

      base64 with mimetype, relative URLs, and absolute URLs.

      For URLs you need to handle cross-domain issues yourself, otherwise the promise will fail.

      Except HTMLCanvasElement and HTMLVideoElement, which have their own format,

      the other Data should be encoded as either jpg, png, bmp or gif.

    ```js // decode file in nodejs, or decode url in web reader.decode('./TestSmall.jpg')).then(r=>{console.log(r)}); // decode blob from input[type="file"] in web reader.decode(inputEl.files[0]).then(r=>{console.log(r)}); ```

    Parameters

    • source: Blob | Buffer | ArrayBuffer | Uint8Array | Uint8ClampedArray | HTMLImageElement | HTMLCanvasElement | HTMLVideoElement | string

    Returns Promise<TextResult[]>

deleteInstance

  • deleteInstance(): void
    • Defined in dbr-wasm.d.ts:74

    Destructor the BarcodeReader object. When the BarcodeReader object is created, it will open up a space in memory that cannot be automatically reclaimed by js garbage collection. You can manually call this method to destruct the BarcodeReader object to release this memory.

    // delete the instance when you don't need it any more.
reader.deleteInstance();

    Returns void
